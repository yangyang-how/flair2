Section 3: Interfaces, Signatures & Orchestrator State Machine

The contracts that bind everything together. If you implement these interfaces, the system works.

Provider Protocols

# providers/base.py

class ReasoningProvider(Protocol):
    """Any LLM that can generate text."""

    name: str  # "gemini" | "kimi" | "openai"

    async def generate_text(
        self,
        prompt: str,
        schema: type[BaseModel] | None = None,
    ) -> str:
        """Generate text, optionally constrained
        to a JSON schema."""
        ...

    async def analyze_content(
        self,
        content: str,    # transcript or description
        prompt: str,
    ) -> str:
        """Analyze content with a prompt."""
        ...


class VideoProvider(Protocol):
    """Any service that generates video."""

    name: str  # "seedance" | "veo"

    async def generate_video(
        self,
        prompt: str,
        duration: int = 6,
    ) -> bytes:
        """Returns video file bytes (mp4)."""
        ...

    async def check_status(
        self,
        job_id: str,
    ) -> VideoJobStatus:
        """Poll async video generation."""
        ...


class VideoJobStatus(BaseModel):
    job_id: str
    status: str   # "processing"|"complete"|"failed"
    video_url: str | None
    error: str | None

Stage Function Signatures

# pipeline/stages/*.py
# Every stage is a pure async function.
# No Redis, no Celery, no side effects.

async def s1_analyze(
    video: VideoInput,
    provider: ReasoningProvider,
) -> S1Pattern:
    """Extract structural patterns from one video."""

def s2_aggregate(
    patterns: list[S1Pattern],
) -> S2PatternLibrary:
    """Merge patterns into ranked library.
    No LLM — pure algorithmic."""

async def s3_generate(
    library: S2PatternLibrary,
    provider: ReasoningProvider,
    feedback: list[VideoPerformance] | None,
) -> list[CandidateScript]:
    """Generate 50 candidate scripts.
    Sequential — deliberate bottleneck."""

async def s4_vote(
    scripts: list[CandidateScript],
    persona_id: str,
    provider: ReasoningProvider,
    feedback: list[VideoPerformance] | None,
) -> PersonaVote:
    """One persona evaluates all scripts."""

def s5_rank(
    votes: list[PersonaVote],
) -> S5Rankings:
    """Aggregate votes into top 10.
    No LLM — pure algorithmic."""

async def s6_personalize(
    script: CandidateScript,
    profile: CreatorProfile,
    provider: ReasoningProvider,
) -> FinalResult:
    """Rewrite script in creator's voice
    + generate video prompt."""

Note: S3 and S4 accept optional feedback — historical performance data for pipeline calibration. When None (first run), they operate without it.

Orchestrator State Machine

Each pipeline run transitions through these states. The orchestrator checks completion conditions and triggers the next stage.

Transition Triggers

PENDING → S1_MAP API calls start_pipeline()
S1_MAP → S2_RED All 100 S1 tasks completed
S2_RED → S3_SEQ Pattern library written
S3_SEQ → S4_MAP All 50 scripts generated
S4_MAP → S5_RED All 100 persona votes in
S5_RED → S6_PERS Top 10 ranked
S6_PERS → COMPLETED All 10 personalized + saved to S3/DynamoDB

Redis Keys (per run)

run:{id}:stage Current state name
run:{id}:s1:done Counter — INCR on each S1 completion
run:{id}:s4:done Counter — INCR on each S4 completion
checkpoint:{id}:{stage} Last completed task index
sse:events:{id} SSE event queue for frontend

Local Runner vs Distributed Runner

Same stage functions, different execution layer:

Local Mode (MVP)

# runner/local_runner.py

async def run_pipeline(
    config: PipelineConfig,
    videos: list[VideoInput],
    provider: ReasoningProvider,
) -> S6Output:
    # S1: analyze all videos sequentially
    patterns = []
    for v in videos:
        p = await s1_analyze(v, provider)
        patterns.append(p)

    # S2: aggregate
    library = s2_aggregate(patterns)

    # S3: generate scripts
    scripts = await s3_generate(
        library, provider, feedback=None
    )

    # S4: vote (sequential, no workers)
    votes = []
    for pid in range(100):
        v = await s4_vote(
            scripts, f"persona_{pid}",
            provider, feedback=None
        )
        votes.append(v)

    # S5: rank
    rankings = s5_rank(votes)

    # S6: personalize top 10
    results = []
    for rs in rankings.top_10:
        script = find_script(scripts, rs)
        r = await s6_personalize(
            script, config.creator_profile,
            provider
        )
        results.append(r)

    return S6Output(
        run_id=config.run_id,
        results=results,
        ...
    )

Distributed Mode

# workers/tasks.py

@celery_app.task(bind=True)
def s1_analyze_task(
    self, run_id: str, video_json: str
):
    video = VideoInput.model_validate_json(
        video_json
    )
    provider = get_provider(run_id)

    result = asyncio.run(
        s1_analyze(video, provider)
    )

    # Write result to Redis
    redis.set(
        f"result:s1:{run_id}:{video.video_id}",
        result.model_dump_json()
    )

    # Increment completion counter
    done = redis.incr(f"run:{run_id}:s1:done")

    # Check if all S1 tasks complete
    if done == 100:
        orchestrator.transition(
            run_id, "S2_RED"
        )

# Same pattern for s4_vote_task, etc.
# The stage function is identical —
# only the wrapper changes.

What Someone Needs To Start Coding

  • To implement a stage: Look at the function signature + input/output models. Write the function. Write a test. Done.
  • To add a provider: Implement the ReasoningProvider or VideoProvider protocol. Register in registry.py. Done.
  • To run locally: python -m app.runner.local_runner — calls stage functions directly.
  • To run distributed: Start Celery workers + Redis. API calls orchestrator. Orchestrator pushes tasks. Same stage functions execute in workers.
