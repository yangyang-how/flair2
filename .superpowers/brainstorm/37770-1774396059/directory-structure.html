Key Design Decisions Reflected

  • runner/local_runner.py — MVP mode. Calls stage functions directly, no Redis/Celery. Same stage code as distributed mode.
  • workers/tasks.py — Thin wrappers only. Each task reads input, calls the pure stage function, writes output. Business logic stays in stages/.
  • models/ — Shared Pydantic models used by all layers. The contracts between stages.
  • prompts/ — Separate from stage logic. Prompt templates are tunable without touching pipeline code.
  • providers/registry.py — Maps provider names (from user selection) to provider instances. The orchestrator looks up "gemini" → gets a GeminiProvider.
